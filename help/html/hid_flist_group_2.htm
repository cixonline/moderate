The File List Editor Display

on the main file list dialog

The file list editor display is intended to appear as close as is practical to the file list display as shown by Ameol to forum users. There are, however three details that differ:

1. Entry Type Indication

An extra column is displayed at the left of the list showing what the type or origin of the entry on that line is. The following characters may be displayed:

#A hidden memo line. The line will not be included in the users' file list and is purely for information to moderators. Memo lines are also displayed in grey.

cNormal file list files.

fFiles imported from the filepool.

uFiles added by upload in this edit session - they will become 'c' lines after the file and moderator's file list have been uploaded.

mFiles added from your binmail directory in this edit session - they will become 'c' lines after the file has been exported and the moderator's file list uploaded.

nFiles added by entering their names from notification messages received by you. They will become 'c' lines after you have refreshed your copy of the file directory and uploaded the moderator's file list.

hComment lines that have been held (that is, temporarily hidden).

Visible comment lines and indented comments are displayed with a blank space in the first column position.

2. Lines Invisible on the Users' File List

Memo lines and file list lines that have been held to hide them from users are displayed. They are displayed in grey to distinguish them from visible lines.

3. Missing File Sizes

In some situations, Moderate cannot determine the size of a file in the file list. It then displays a '?' character in place of the size in its display. This happens for:

All filepool file entries.

Files that have been added in an earlier editor session if you haven't blinked in between times - Moderate does not know in this case where the files have been added from, thus it cannot check that location for their sizes.

Files that are actually missing from the toic file directory. Note that such files will not be included in the users' file list when CIX creates it from the moderator's list and file directory.
